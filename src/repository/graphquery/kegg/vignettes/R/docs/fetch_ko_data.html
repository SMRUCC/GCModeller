fetch_ko_data {kegg_api} R Documentation

Fetch and Download Data for a Specific KO Entry

Description

An internal function that downloads gene ID associations for a specific KEGG Orthology (KO) identifier and optionally filters them by species.

Usage

fetch_ko_data(db, ko.id, species,
    seqtype = 'c'('ntseq', 'aaseq'),
    download.seqs = TRUE);

Arguments

db

A local repository connection to the target KEGG genes database.

ko.id

A character string specifying the KEGG Orthology identifier (e.g., \code{"K00001"}).

species

A character vector of KEGG organism species codes for filtering. Default is \code{NULL} (no filtering). Only genes from the specified organisms will be downloaded.

seqtype

The type of FASTA sequence data to download. Must be either \code{"ntseq"} (gene nucleotide sequence) or \code{"aaseq"} (protein amino acid sequence). Default is \code{"ntseq"}.

Details

This function performs the following steps: \enumerate{ \item Checks if a cached gene ID list exists for the KO entry \item If not cached, downloads the gene ID list from the KEGG REST API \item Parses the gene IDs into species code and gene locus tag components \item Filters genes by the specified species (if provided) \item Triggers the download of FASTA sequences for the filtered genes } The KEGG API returns gene associations in the format \code{ko:K00001\teco:b0001}, where the first column is the KO identifier and the second column is the gene identifier (formatted as \code{species:gene_id}).

xieguigang

Value

Returns \code{NULL} invisibly.
